Updated toggleFailed for both files:
```javascript
function toggleFailed(checkbox) {
  const tasteGroup = document.getElementById('taste-group');
  const tasteInput = document.getElementById('taste');
  const tasteDisplay = document.getElementById('taste-display');
  const requiredMsg = document.getElementById('taste-required-msg');

  if (checkbox.checked) {
    tasteInput.value = '1';
    tasteInput.dataset.touched = 'true';
    tasteDisplay.textContent = '1.0';
    tasteGroup.classList.add('touched');
    tasteGroup.style.pointerEvents = 'none';
    if (requiredMsg) requiredMsg.classList.remove('visible');
  } else {
    tasteInput.value = '7.0';
    tasteInput.dataset.touched = 'false';
    tasteDisplay.textContent = '\u2014';
    tasteGroup.classList.remove('touched');
    tasteGroup.style.pointerEvents = 'auto';
    tasteGroup.style.opacity = '';
  }
}
```

Note the `tasteGroup.style.opacity = ''` in the else branch — this clears any inline opacity that the old toggleFailed might have set, letting the CSS class take over. In the checked branch, we don't set inline opacity because the `.touched` class sets opacity to 1, and then `pointer-events: none` handles the disabled look. Actually, the Failed Shot checked state needs to LOOK dimmed (opacity 0.4) even though `.touched` is added. The `.touched` class adds opacity 1, which conflicts. 

**Revised approach for Failed Shot checked state:** When checked, add `.touched` (for submit gate to pass), but also set inline `opacity: 0.4` to override the CSS class. When unchecked, clear inline opacity and remove `.touched`:

```javascript
function toggleFailed(checkbox) {
  const tasteGroup = document.getElementById('taste-group');
  const tasteInput = document.getElementById('taste');
  const tasteDisplay = document.getElementById('taste-display');
  const requiredMsg = document.getElementById('taste-required-msg');

  if (checkbox.checked) {
    tasteInput.value = '1';
    tasteInput.dataset.touched = 'true';
    tasteDisplay.textContent = '1.0';
    tasteGroup.classList.add('touched');
    tasteGroup.style.opacity = '0.4';
    tasteGroup.style.pointerEvents = 'none';
    if (requiredMsg) requiredMsg.classList.remove('visible');
  } else {
    tasteInput.value = '7.0';
    tasteInput.dataset.touched = 'false';
    tasteDisplay.textContent = '\u2014';
    tasteGroup.classList.remove('touched');
    tasteGroup.style.opacity = '';
    tasteGroup.style.pointerEvents = 'auto';
  }
}
```

This preserves the exact visual behavior of Failed Shot (dimmed + disabled) while setting `data-touched='true'` so the submit gate passes.
