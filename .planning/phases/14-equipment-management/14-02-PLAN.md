**Add grinder routes to `app/routers/equipment.py`:**

`POST /equipment/grinders` — Create grinder. Form fields:
- `name: str = Form(...)` (required)
- `dial_type: str = Form("stepless")` — "stepped" or "stepless"
- `step_size: str = Form("")` — parse to float if stepped, else None
- `min_value: str = Form("")` — parse to float or None
- `max_value: str = Form("")` — parse to float or None
- Create Grinder object, commit. If htmx, return the grinder card partial. Otherwise POST-redirect-GET to /equipment.

`GET /equipment/grinders/{grinder_id}/edit` — Return the edit form partial (htmx response for modal content).

`POST /equipment/grinders/{grinder_id}` — Update grinder. Same form fields as create. Find by ID, update fields, commit. Redirect to /equipment.

**Create `app/templates/equipment/_grinder_card.html`:**
- Card displaying grinder name, dial type ("Stepped" or "Stepless"), and range info
- If stepped: show "Steps: {min} – {max}, step {step_size}"
- If stepless: show "Range: {min} – {max}" (only if min/max set)
- Edit button that opens the form modal (use htmx: `hx-get="/equipment/grinders/{id}/edit" hx-target="#equipment-modal-content" hx-swap="innerHTML"` and triggers modal open)
- If retired, show a visual indicator (e.g., muted/grayed out with "Retired" badge)

**Create `app/templates/equipment/_grinder_form.html`:**
- Modal form for create/edit grinder. If `grinder` variable exists, pre-fill for edit mode.
- Name input (required, text)
- Dial type: two radio buttons or toggle — "Stepped" / "Stepless"
- Conditional fields (shown/hidden by JS based on dial_type):
  - When "Stepped": show step_size, min_value, max_value inputs
  - When "Stepless": show min_value, max_value inputs (step_size hidden)
- All numeric fields are type="number" with step="any"
- Submit button: "Add Grinder" (create) or "Save Changes" (edit)
- Form action: `POST /equipment/grinders` (create) or `POST /equipment/grinders/{id}` (edit)
- Include simple JS to toggle stepped/stepless field visibility

**Update equipment/index.html Grinders section:**
- Include add button that opens the create form modal
- Loop over grinders and render `_grinder_card.html` for each
- Add a modal container div (e.g., `<div id="equipment-modal" class="modal">`) at page level if not already present, with `<div id="equipment-modal-content">` inside

**Modal pattern:** Use a simple CSS modal (overlay + centered card). Add `.modal` and `.modal.open` styles to main.css. Open/close via JS: clicking "Add" sets modal to `.open`, close button or overlay click removes `.open`. The modal content is loaded via htmx into `#equipment-modal-content`.
