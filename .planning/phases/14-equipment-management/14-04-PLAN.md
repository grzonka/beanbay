**Add setup routes to `app/routers/equipment.py`:**

`GET /equipment/setups/new` — Start the wizard. Query all active (non-retired) brewers, grinders, papers, water recipes. Return the wizard template.

`POST /equipment/setups` — Create brew setup. Form fields:
- `name: str = Form(...)` (required)
- `brewer_id: str = Form(...)` (required)
- `grinder_id: str = Form(...)` (required)
- `paper_id: str = Form("")` (optional — paper can be skipped)
- `water_recipe_id: str = Form(...)` (required)
- Determine `brew_method_id`: look up the selected brewer, use its first associated method. If the brewer has multiple methods, accept a `method_id` form field too. If brewer has no methods, create setup without method (or use a default).
- Create BrewSetup, commit. Redirect to /equipment.

`GET /equipment/setups/{setup_id}/edit` — Return the wizard template pre-filled with existing setup data for editing.

`POST /equipment/setups/{setup_id}` — Update brew setup. Same fields, update existing record.

**Create `app/templates/equipment/_setup_wizard.html`:**

Implement as a **single-page wizard with step sections** (not multi-page). All steps are in one HTML form, with JS showing/hiding steps:

Step 1 — "Pick a Brewer":
- Radio buttons or selectable cards for each active brewer
- Each shows brewer name + method badges
- "Add New Brewer" small link/button that opens the brewer create modal (from Plan 02)

Step 2 — "Pick a Grinder":
- Radio buttons or selectable cards for each active grinder
- Each shows grinder name + dial type info
- "Add New Grinder" link

Step 3 — "Pick a Paper/Filter" (optional):
- Radio buttons or selectable cards for each active paper
- "None / Skip" option (a radio that sends empty string)
- "Add New Paper" link

Step 4 — "Pick a Water Recipe":
- Radio buttons or selectable cards for each active water recipe
- Each shows name + mineral summary if available
- "Add New Water Recipe" link

Step 5 — "Name Your Setup":
- Text input for setup name (required)
- Suggested auto-name: "{brewer_name} + {grinder_name}" (JS auto-fills, user can override)
- Summary of selected components below the name field

Navigation: "Back" and "Next" buttons at each step. Step indicator (dots or numbered steps). Final step has "Create Setup" submit button.

JS logic:
- Track current step index (0-4)
- "Next" validates current step selection, shows next step div, hides current
- "Back" shows previous step
- On submit, all selected IDs + name are submitted as one form POST

For edit mode: pre-select the existing component in each step, pre-fill name.

**Important:** The "Add New" links in each step should open the same equipment create modals from Plans 02-03. After the modal submits (htmx), the newly created item should appear in the wizard step's selection list. This can be done with htmx: the create form targets a partial refresh of the wizard step's options list, or simply with a page reload approach where the wizard state is preserved via URL params/hidden fields.

For simplicity, OpenCode may choose to implement the "Add New" as a link that opens a small inline form (accordion within the step) rather than reusing the full modal, if that's cleaner for the wizard flow.

Touch targets: 48px+ for all radio/card selections and buttons.
