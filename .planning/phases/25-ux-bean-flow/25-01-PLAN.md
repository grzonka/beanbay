**insights.py — Add bean_id query param with cookie fallback:**

Modify the `insights_page` route (GET /insights):

1. Add `bean_id: Optional[str] = None` query parameter
2. Logic:
   - If `bean_id` is provided, look up that bean
   - Else fall back to `_get_active_bean(request, db)` (cookie)
   - If still no bean, do NOT redirect to /beans — instead render insights with `bean=None` and show a "pick a bean" prompt
3. Query all beans for the dropdown: `beans = db.query(Bean).order_by(Bean.name).all()`
4. Pass `beans` and `selected_bean_id` to template context alongside existing vars
5. If no bean selected (and no measurements to show), render a minimal page with just the dropdown and a message

Import `Optional` from typing if not already imported.

**insights/index.html — Add bean picker and handle no-bean state:**

Add a bean picker dropdown at the top of the page, right after the heading:

```html
<!-- Bean selector -->
<div class="card bg-base-200 border border-base-300 mb-4">
  <div class="card-body p-4">
    <div class="text-sm font-semibold text-base-content/70 mb-1 uppercase tracking-wider">Bean</div>
    <select class="select select-bordered w-full min-h-12"
            onchange="window.location.href='/insights?bean_id=' + this.value">
      <option value="" disabled {% if not active_bean %}selected{% endif %}>Pick a bean…</option>
      {% for b in beans %}
      <option value="{{ b.id }}" {% if active_bean and b.id == active_bean.id %}selected{% endif %}>{{ b.name }}</option>
      {% endfor %}
    </select>
  </div>
</div>
```

Wrap the existing content (bean header card, convergence, charts, etc.) in a `{% if active_bean %}...{% else %}` block:

```html
{% if active_bean %}
  {# existing insights content #}
{% else %}
  <div class="text-center py-8 text-base-content/50">
    <p class="text-sm">Select a bean above to see optimization insights.</p>
  </div>
{% endif %}
```
