**analytics.py — Add bean_id filter:**

1. Add `bean_id: Optional[str] = None` query parameter to `analytics_page` route.
2. Add `from typing import Optional` import.
3. Query all beans for the dropdown: `beans = db.query(Bean).order_by(Bean.name).all()`
4. Modify `_compute_stats` to accept an optional `bean_id` parameter:
   - If `bean_id` is provided, filter: `db.query(Measurement).filter(Measurement.bean_id == bean_id).all()`
   - If not, query all measurements (current behavior)
   - When filtering by bean, `total_beans` should be 1, `best_bean_name` is that bean's name
5. Pass to template: `beans`, `selected_bean_id` (the bean_id param, or `""` for "all"), and the selected bean object if applicable.
6. When a bean_id is selected, `_compute_comparison` is unnecessary (it's a cross-bean view) — pass `comparison=[]` when filtering by bean.

**analytics/index.html — Add bean picker dropdown:**

Add a bean filter dropdown between the heading and the stats card:

```html
<!-- Bean filter -->
<div class="card bg-base-200 border border-base-300 mb-4">
  <div class="card-body p-4">
    <div class="text-sm font-semibold text-base-content/70 mb-1 uppercase tracking-wider">Filter by Bean</div>
    <select class="select select-bordered w-full min-h-12"
            onchange="window.location.href='/analytics' + (this.value ? '?bean_id=' + this.value : '')">
      <option value="" {% if not selected_bean_id %}selected{% endif %}>All beans</option>
      {% for b in beans %}
      <option value="{{ b.id }}" {% if selected_bean_id == b.id %}selected{% endif %}>{{ b.name }}</option>
      {% endfor %}
    </select>
  </div>
</div>
```

When a bean is selected:
- Stats card shows per-bean stats (existing `_stats_card.html` template works — just receives filtered data)
- Hide or replace the "Best Recipes by Bean" comparison section with a single "Best Recipe" card showing that bean's best recipe
- Or conditionally hide the comparison table: `{% if comparison %}...{% endif %}` (since we pass `comparison=[]` when filtering)

When "All beans" is selected:
- Show everything as before (aggregate stats + cross-bean comparison)

**_comparison_table.html — No changes needed** if we pass `comparison=[]` when filtering by bean. The existing `{% if not comparison %}` guard will show "No beans with shots yet." — but that's wrong when filtering a single bean. Better: wrap the comparison section in the parent template with `{% if not selected_bean_id %}` to hide it entirely when filtering.

- Add `test_analytics_all_beans_default`: GET /analytics without params still shows aggregate (existing behavior)
- Add `test_analytics_bean_filter_invalid_id`: GET /analytics?bean_id=nonexistent shows empty stats gracefully
