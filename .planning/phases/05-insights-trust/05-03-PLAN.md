  **Fix 1 — Recommend crash (blocker):**

  In `optimizer.py`, method `_recommend` (the inner function inside `recommend()`, around line 190-204), add `campaign.clear_cache()` immediately before `campaign.recommend(batch_size=1)`.

  Current code (lines 190-194):
  ```python
  def _recommend():
      campaign = self.get_or_create_campaign(bean_id, overrides)
      with self._lock:
          rec_df = campaign.recommend(batch_size=1)
          self._save_campaign_unlocked(bean_id)
  ```

  Change to:
  ```python
  def _recommend():
      campaign = self.get_or_create_campaign(bean_id, overrides)
      with self._lock:
          campaign.clear_cache()
          rec_df = campaign.recommend(batch_size=1)
          self._save_campaign_unlocked(bean_id)
  ```

  **Why:** BayBE 0.14.2 sets `allow_recommending_already_recommended = UNSPECIFIED` for hybrid search spaces. On 2nd+ calls, the cache guard in `Campaign.recommend()` (line 513-518 of campaign.py) tries to evaluate `_cached_recommendation` as bool but hits `UNSPECIFIED.__bool__()` → `NotImplementedError`. Calling `clear_cache()` resets `_cached_recommendation` to `None`, so the walrus operator short-circuits before reaching the UNSPECIFIED field.

  **Fix 2 — switch_after=5:**

  In `optimizer.py`, line 123, change:
  ```python
  recommender = TwoPhaseMetaRecommender(recommender=BotorchRecommender())
  ```
  to:
  ```python
  recommender = TwoPhaseMetaRecommender(recommender=BotorchRecommender(), switch_after=5)
  ```

  **Why:** The default `switch_after=1` switches to Bayesian mode after just 1 measurement. With only 1 data point, the GP model is essentially uninformed — the "Bayesian optimization" label is misleading. Setting `switch_after=5` keeps the optimizer in random exploration for the first 5 shots, building a meaningful dataset before switching.

  **Fix 3 — Add "bayesian_early" sub-phase in `get_recommendation_insights`:**

  In `get_recommendation_insights()` (around lines 298-304), update the Bayesian phase logic to distinguish early Bayesian from confident Bayesian:

  ```python
  else:
      if shot_count < 8:

Since `switch_after=5`, the random phase now covers shots 0-4. The "bayesian_early" / "Learning" sub-phase covers shots 5-7. Full "Bayesian optimization" starts at 8+.

  The full updated else block should be:
  ```python

          recent_best = max(taste_values[-3:])

              explanation = (
                  "Zeroing in — recent shots are improving. "

              )

              )
  ```
