**conftest.py — Update fixtures:**

1. Replace the `optimizer_service` fixture. The new fixture must create a session factory that yields the test `db_session` (for isolation with rollback). OptimizerService uses `session = self._session_factory()` then `try/finally session.close()`. So the factory must return a session object that works with this pattern BUT shares the same transaction as `db_session` for test isolation.

The simplest approach: create a factory callable that returns the existing `db_session`. Since `db_session` uses a nested transaction (connection + transaction), returning it directly means OptimizerService's `session.close()` calls won't actually close the underlying connection (the fixture manages that). BUT if OptimizerService calls `session.commit()`, it will commit within the test's transaction scope.

```python
@pytest.fixture()
def optimizer_service(db_session) -> OptimizerService:
    """OptimizerService with test DB session factory."""
    def _test_session_factory():
        return db_session
    return OptimizerService(_test_session_factory)
```

This works because:
- OptimizerService calls `session = self._session_factory()` → gets db_session
- OptimizerService calls `session.commit()` → commits within the test transaction
- OptimizerService calls `session.close()` → no-op since fixture manages lifecycle
- After test, fixture rolls back the transaction → all data cleaned up

2. Keep the `tmp_campaigns_dir` fixture — it's still needed for migration tests and for `test_brew_multimethod.py` legacy migration tests. But DON'T use it in the `optimizer_service` fixture.
