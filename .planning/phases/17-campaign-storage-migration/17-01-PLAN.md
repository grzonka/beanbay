  <action>
Create `app/services/migration.py` with two functions:

**`migrate_campaigns_to_db(session_factory, campaigns_dir: Path) -> int`:**
- If `campaigns_dir` does not exist, return 0
- Open a single session from `session_factory()` as context manager
- Iterate over sorted `campaigns_dir.glob("*.json")` files
- For each JSON file:
  - `campaign_key = json_file.stem`
  - Check if `CampaignState` row with this `campaign_key` already exists → skip if so (idempotency)
  - Read `campaign_json` from the JSON file
  - Read `bounds_fingerprint` from `{campaign_key}.bounds` file if it exists (strip whitespace), else None
  - Read `transfer_metadata` from `{campaign_key}.transfer` file if it exists (json.loads), else None
  - Create `CampaignState(campaign_key=campaign_key, campaign_json=campaign_json, bounds_fingerprint=fingerprint, transfer_metadata=transfer_meta)`
  - Add to session
  - Increment counter
- Commit if any rows were added
- Return count of migrated campaigns
- Wrap file reads in try/except for robustness (log warning on corrupt files, skip them)

**`migrate_pending_to_db(session_factory, data_dir: Path) -> int`:**
- Look for `data_dir / "pending_recommendations.json"`
- If file doesn't exist, return 0
- Read and parse JSON (dict of {rec_id: rec_data})
- Handle JSONDecodeError/OSError gracefully → return 0
- Open a single session from `session_factory()` as context manager
- For each rec_id, rec_data pair:
  - Check if `PendingRecommendation` row with this `recommendation_id` already exists → skip (idempotency)
  - Create `PendingRecommendation(recommendation_id=rec_id, recommendation_data=rec_data)`
  - Add to session
  - Increment counter
- Commit if any rows were added
- Return count

Both functions use `session_factory` as a context manager: `with session_factory() as session:`. This matches the pattern that will be used by OptimizerService in Plan 02.

Import `logging` and create a module logger: `logger = logging.getLogger(__name__)`. Log info for each migrated file and warnings for skipped/corrupt files.
  </action>
