    - Implementation approach (client-side with localStorage):
      1. Create `app/static/js/hints.js` with a `ParamHintManager`:
         - Tracks which params the user has seen via `localStorage.getItem('param_hints_seen')` (JSON set of param names)
         - For each param in the recommendation, if not in the seen set, shows a hint tooltip/message below the param value
         - When hint is dismissed (click X or auto-dismiss after first view), adds to seen set
      2. In `recommend.html`, add `data-param-hint="..."` attributes to each param display element with the hint text.
      3. Include `<script src="/static/js/hints.js"></script>` in the template.
    - Hint texts per param (keep brief):
      - `preinfusion_time`: "Hold at low pressure for this duration before ramping up"
      - `preinfusion_pressure`: "Pressure during the pre-infusion phase (bar)"
      - `brew_pressure`: "Target pressure during the main extraction (bar)"
      - `pressure_profile`: "How pressure changes during extraction"
      - `flow_rate`: "Water flow rate through the puck (ml/s)"
      - `bloom_pause`: "Pause to let grounds bloom before continuing"
      - `temp_profile`: "How temperature changes during extraction"
      - `brew_mode`: "Whether to prioritize pressure or flow control"
      - `saturation`: "Pre-wet the puck at low flow rate before extraction"
    - Store hint texts in a `PARAM_HINTS` dict in `brew.py` and pass to template context.

    **2b. "New" badge after campaign rebuild:**
    - When a campaign is rebuilt with new params (Plan 02's `accept_rebuild`), the recommendation should show a "new" badge on params that weren't in the previous campaign.
    - Implementation:
      1. In `optimizer.py`'s `accept_rebuild`, store the list of new param names in the pending recommendation or campaign metadata.
      2. In `brew.py`'s `trigger_recommend`, after a rebuild, pass `new_params` (set of param names) to the template context.
      3. In `recommend.html`, for each param: if `param_name in new_params`, show a `<span class="badge badge-accent badge-sm">new</span>` next to the value.
      4. The "new" badge disappears after the first recommendation that uses the new params. Track this by clearing the `new_params` metadata after the first recommendation post-rebuild.
    - If this is too complex for this plan, a simpler approach: `new_params` is derived by comparing the current recommendation's param keys to a "baseline" set (the 4 Tier 1 params). Any param beyond Tier 1 that appears for the first time in this bean's recommendation gets a "new" badge. Store seen params in localStorage alongside hints.

    **2c. Flat ordered param display:**

      6. Legacy: Preinfusion pressure pct (only for old campaigns)
