<task type="auto">
  <name>Task 1: Dashboard route and data assembly</name>
  <files>app/main.py, app/templates/base.html</files>
  <action>
Modify the root route in `app/main.py` to render a dashboard for returning users instead of redirecting to `/beans`.

**Root route changes (`/`):**
- Keep the bean_count == 0 → welcome.html branch (unchanged)
- When beans exist, query and assemble dashboard data, then render `home.html`

**Data to assemble:**
1. **Aggregate stats** (reuse pattern from `analytics.py._compute_stats`):
   - `total_brews`: `db.query(Measurement).count()`
   - `total_beans`: `db.query(Bean).count()`
   - `avg_taste`: average taste of non-failed measurements, rounded to 1 decimal
   - `best_taste` + `best_bean_name`: highest taste score and which bean achieved it
   - If total_brews == 0, set avg_taste/best_taste to None

2. **Recent brews** (last 5 measurements):
   - Query `Measurement` joined with `Bean` and optionally `BrewSetup`, ordered by `created_at desc`, limit 5
   - Build simple dicts: `{id, bean_name, taste, grind_setting, is_failed, is_manual, created_at, brew_method, brew_setup_name}`
   - For `brew_method`: if measurement has a brew_setup with a brew_method, use `setup.brew_method.name`, else default to `"espresso"`
   - For `brew_setup_name`: use `setup.name` if setup exists, else None

3. **Active bean info:**
   - Use `_get_active_bean` from `beans.py` (already imported pattern)
   - If active bean exists, also query its shot count and best non-failed taste

4. **Equipment counts** (simple counts for context):
   - `setup_count`: `db.query(BrewSetup).filter(BrewSetup.is_retired == False).count()`

Pass all data to template: `stats`, `recent_brews`, `active_bean`, `active_bean_shots`, `active_bean_best`, `setup_count`

**Import additions in main.py:**
- `from app.models.measurement import Measurement`
- `from app.models.brew_setup import BrewSetup`
- `from app.routers.beans import _get_active_bean`
- `from sqlalchemy import func`

**base.html changes:**
- Change both brand `href="/beans"` links to `href="/"` (sidebar brand link on line 46 and mobile navbar brand link on line 29)

Do NOT add a "Home" nav link to the sidebar — the brand link IS the home link (same pattern as most web apps).

<task type="auto">
  <name>Task 2: Dashboard template</name>
  <files>app/templates/home.html</files>
  <action>
Create `app/templates/home.html` extending `base.html`. Follow established patterns from analytics, insights, and welcome templates.

**Layout structure (top to bottom):**

1. **Page header:**
   ```html
   <div class="flex items-center justify-between mb-6 min-h-12">
     <h1 class="text-xl font-bold">Dashboard</h1>
   </div>
   ```

2. **Stats grid** (reuse `stats-grid` CSS class from analytics):
   A 2-column grid with 4 stat tiles:
   - Total Brews (`stats.total_brews`) — label "Total brews"
   - Beans Tracked (`stats.total_beans`) — label "Beans tracked"
   - Avg Taste (`stats.avg_taste` or "—") — label "Avg taste"
   - Best Taste (`stats.best_taste` or "—") in `text-primary` — label "Best taste", with `stats.best_bean_name` as subtitle if present

   Wrap in: `<div class="card bg-base-200 border border-base-300 mb-4"><div class="card-body p-4">`
   Each tile: `<div class="text-center p-3 bg-base-300/50 rounded-lg">`

3. **Active bean card** (only if `active_bean` is set):
   ```
   Card with:
   - Bean name (font-semibold) + "Active" badge (badge-primary)
   - Shot count: "X brews recorded" (text-sm text-base-content/50)
   - Best taste for this bean (text-2xl font-bold text-primary) on right side
   - "Let's Brew" button (btn btn-primary) linking to /brew
   ```
   Follow the pattern from insights/index.html bean header card.

   If no active bean: show a subtle prompt "No bean selected — pick one to start brewing" with link to /beans.

4. **Recent brews** (only if `recent_brews` is not empty):
   Card with heading "Recent Brews":
   - List of up to 5 recent brews as compact rows (similar to shot_row but simpler, no checkbox/delete):
     ```
     Each row: date (text-sm text-base-content/70) | taste score (text-lg font-bold) | bean name (truncate) | method badge (if not espresso)
     Failed brews: taste in text-error, "Failed" badge
     ```
   - "View All History →" link at bottom (text-sm, links to /history)

   If no brews exist: "No brews yet — start your first brew to see results here" with brew CTA.

5. **Quick actions** (bottom):
   ```html
   <div class="flex flex-col gap-3">
     <a href="/brew" class="btn btn-primary w-full min-h-14 rounded-xl text-base">Let's Brew</a>
     <div class="grid grid-cols-2 gap-3">
       <a href="/beans" class="btn btn-ghost min-h-12 border border-base-300">Beans</a>
       <a href="/equipment" class="btn btn-ghost min-h-12 border border-base-300">Equipment</a>
     </div>
   </div>
   ```

**Empty states:**
- If stats.total_brews == 0 but beans exist: show stats grid with zeroes, skip recent brews section, show "You have beans but no brews yet — time to start!" message with brew CTA.

**Do NOT:**
- Add Chart.js to the dashboard (keep it lightweight; charts are on Insights/Analytics)
- Duplicate the full shot row template (keep recent brews simple and compact)
- Add any custom CSS classes — use existing daisyUI + Tailwind + stats-grid from input.css
