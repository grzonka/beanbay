<task type="auto">
  <name>Task 1: Fix wizard CSS — only show active step</name>
  <files>app/static/css/input.css</files>
  <action>
  **The root cause bug:** The `.wizard-step-content` class is used in the wizard template (5 divs, one per step) and the JavaScript toggles `.active` class on them, but there are NO CSS rules for `.wizard-step-content` in `input.css`. This means all 5 steps are visible simultaneously — the wizard is completely broken visually.

  **Add these rules to `app/static/css/input.css`** in the `@layer components` section, right after the existing `.wizard-option-card.selected` rule (around line 323):

  ```css
  /* Wizard step content panels — only active step is visible */
  .wizard-step-content {
    display: none;
  }

  .wizard-step-content.active {
    display: block;
  }
  ```

  Also add missing layout styles for the wizard option containers:

  ```css
  /* Wizard options container — vertical stack of radio cards */
  .wizard-options {
    display: flex;
    flex-direction: column;
    gap: 0.75rem;
    margin-bottom: 1rem;
  }

  /* Wizard option card text elements */
  .wizard-option-name {
    font-weight: 600;
    font-size: 0.95rem;
  }

  .wizard-option-meta {
    color: oklch(70% 0.02 55);
    font-size: 0.8rem;
    margin-top: 0.25rem;
  }

  /* "Add first" links at bottom of empty steps */
  .wizard-add-link {
    display: inline-block;
    margin-top: 0.75rem;
    font-size: 0.875rem;
    color: oklch(72% 0.123 63);
  }

  .wizard-add-link:hover {
    text-decoration: underline;
  }
  ```

  **After editing input.css, rebuild CSS.** Check Makefile for the command — likely something like:
  ```bash
  npx @tailwindcss/cli -i app/static/css/input.css -o app/static/css/main.css --minify
  ```
  or run `make css` or similar.

<task type="auto">
  <name>Task 2: Improve wizard UX — inline validation, polished confirmation, post-submit flow</name>
  <files>app/templates/equipment/_setup_wizard.html</files>
  <action>
  **Replace `alert()` validation with inline error messages** in the JavaScript section:

  1. Replace the `validateStep()` function. Instead of `alert()`, show an inline error message below the step heading. Add an error div in each step content area (or use a single floating error that moves with the current step):

  For each step that requires validation (0=Brewer, 1=Grinder, 3=Water, 4=Name), add an error element:
  ```html
  <div class="text-error text-sm mt-2 hidden" id="step-0-error">Please select a brewer to continue.</div>
  ```

  Update `validateStep()` to show/hide these error divs:
  ```javascript
  function validateStep(n) {
    // Clear all errors first
    document.querySelectorAll('[id^="step-"][id$="-error"]').forEach(function(el) {
      el.classList.add('hidden');
    });

    if (n === 0 && !getSelectedValue('brewer_id')) {
      document.getElementById('step-0-error').classList.remove('hidden');
      return false;
    }
    if (n === 1 && !getSelectedValue('grinder_id')) {
      document.getElementById('step-1-error').classList.remove('hidden');
      return false;
    }
    if (n === 3 && !getSelectedValue('water_recipe_id')) {
      document.getElementById('step-3-error').classList.remove('hidden');
      return false;
    }
    if (n === 4) {
      var nameInput = document.getElementById('setup-name');
      if (!nameInput.value.trim()) {
        document.getElementById('step-4-error').classList.remove('hidden');
        nameInput.focus();
        return false;
      }
    }
    return true;
  }
  ```

  2. **Enhance Step 5 (Name/Confirm) to feel like a proper confirmation step.** The summary card already exists — make it more prominent. Change the step 5 heading to "Confirm Your Setup" and make the summary card feel like a final review:

  ```html
  <div class="wizard-step-content" id="step-4">
    <h2 class="text-xl font-semibold mb-4">Confirm Your Setup</h2>
    <p class="text-base-content/60 text-sm mb-4">Review your selections and give this setup a name.</p>

    <div class="form-control mb-6">
      <label class="label"><span class="label-text">Setup Name</span></label>
      <input type="text" id="setup-name" name="name" ...>
      ...
    </div>

    <div class="card bg-base-200 border border-base-300 mb-6" id="setup-summary-preview">
      <div class="card-body p-4">
        <div class="font-semibold text-base mb-3">Setup Summary</div>
        <!-- brewer, grinder, filter, water rows -->
      </div>
    </div>
    <div class="text-error text-sm mt-2 hidden" id="step-4-error">Please give your setup a name.</div>
  </div>
  ```

  3. **Fix the XSS-vulnerable JS object literals.** The equipment name maps use unescaped Jinja2 strings in JS contexts. Use `|tojson` filter:
  ```javascript
  var BREWERS = {
    {% for brewer in brewers %}{{ brewer.id | tojson }}: {{ brewer.name | tojson }}{% if not loop.last %},{% endif %}{% endfor %}
  };
  ```
  Apply this to all four maps (BREWERS, GRINDERS, PAPERS, WATER_RECIPES).

  4. **Add a loading state to the submit button** to prevent double-clicks and provide feedback:
  ```html
  <button type="submit" id="btn-submit" class="btn btn-primary min-h-12 flex-1" style="display:none;"
    onclick="this.classList.add('loading'); this.disabled = true;">
    {% if edit_mode %}Save Setup{% else %}Create Setup{% endif %}
  </button>
  ```

  5. **Improve the submit button text for create mode** to be more inviting:
  Change from "Create Setup" to "Create Setup & Start Brewing →" for create mode (not edit mode).

  6. **Clear errors when a selection is made.** Add to each radio's `onchange` handler (or add a delegated listener):
  ```javascript
  // Add to showStep or as a delegated listener
  document.getElementById('wizard-form').addEventListener('change', function() {
    document.querySelectorAll('[id^="step-"][id$="-error"]').forEach(function(el) {
      el.classList.add('hidden');
    });
  });
  ```
