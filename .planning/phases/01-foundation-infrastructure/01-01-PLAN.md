<task type="auto">
  <name>Task 2: Create SQLAlchemy models and Alembic migrations</name>
  <files>

  <action>
    Create app/models/bean.py with Bean model:
    - __tablename__ = "beans"
    - id: String, primary_key, default=lambda: str(uuid.uuid4())
    - name: String, nullable=False
    - roaster: String, nullable=True
    - origin: String, nullable=True
    - created_at: DateTime, server_default=func.now()
    - measurements: relationship("Measurement", back_populates="bean", cascade="all, delete-orphan")

    Create app/models/measurement.py with Measurement model:
    - __tablename__ = "measurements"
    - id: Integer, primary_key, autoincrement=True
    - bean_id: String, ForeignKey("beans.id"), nullable=False, index=True
    - recommendation_id: String, nullable=True, unique=True (idempotency token)
    - 6 BayBE parameter columns: grind_setting (Float), temperature (Float), preinfusion_pct (Float), dose_in (Float), target_yield (Float), saturation (String) — all nullable=False
    - taste: Float, nullable=False
    - extraction_time: Float, nullable=True
    - is_failed: Boolean, default=False
    - notes: String, nullable=True
    - 6 flavor columns (all Float, nullable=True): acidity, sweetness, body, bitterness, aroma, intensity
    - created_at: DateTime, server_default=func.now()
    - bean: relationship("Bean", back_populates="measurements")

    Update app/models/__init__.py to import Bean and Measurement so they're registered with Base.

    Initialize Alembic:
    - Run `alembic init migrations` to scaffold
    - Edit alembic.ini: set sqlalchemy.url to empty (we override in env.py), set script_location = migrations
    - Edit migrations/env.py:
      - Import Base from app.database and import app.models (so models are registered)
      - Import settings from app.config
      - Set target_metadata = Base.metadata
      - In run_migrations_online(), set url to settings.effective_database_url
      - Set render_as_batch=True in context.configure() — CRITICAL for SQLite (ALTER TABLE limitations)
    - Auto-generate initial migration: `alembic revision --autogenerate -m "initial schema"`
    - Apply migration: `alembic upgrade head`
    - Verify tables exist: `python -c "from app.database import engine; from sqlalchemy import inspect; print(inspect(engine).get_table_names())"`

    IMPORTANT: The data_dir (./data by default) must exist before running migrations. Create it if needed.
  </action>
  <verify>

    Run: python -c "from app.database import engine; from sqlalchemy import inspect; i = inspect(engine); print(i.get_table_names()); assert 'beans' in i.get_table_names(); assert 'measurements' in i.get_table_names(); print('Tables OK')"
    Run: python -c "from app.models import Bean, Measurement; print(Bean.__tablename__, Measurement.__tablename__)"
    All three should succeed.
  </verify>
