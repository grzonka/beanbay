Create the new module `app/services/parameter_registry.py` with the following components:

**1. `PARAMETER_REGISTRY` dict** — maps method name to parameter definitions. Structure for each method:
```python
PARAMETER_REGISTRY = {
    "espresso": {
        "core": [...],
        "advanced": [...],
        "categorical": [...],  # only espresso has this (saturation — deprecated but kept for backward compat)
    },
    "pour-over": { "core": [...], "advanced": [...] },
    "french-press": { "core": [...], "advanced": [...] },
    "aeropress": { "core": [...], "advanced": [...] },
    "turkish": { "core": [...], "advanced": [...] },
    "moka-pot": { "core": [...], "advanced": [...] },
    "cold-brew": { "core": [...], "advanced": [...] },
}
```

Each parameter entry is a dict with keys:
- `name`: str — parameter name
- `type`: str — "continuous", "discrete", or "categorical"
- `default_range`: tuple[float, float] — for continuous params
- `values`: list — for categorical/discrete params
- `rounding`: float — rounding step for continuous params
- `unit`: str (optional) — display unit
- `requires`: str (optional) — capability condition string

**CRITICAL — Espresso backward compatibility.** The current espresso params are:
- `grind_setting` (15, 25) rounding 0.5
- `temperature` (86, 96) rounding 1.0 — NO requires condition (backward compat: always included by default)
- `preinfusion_pct` (55, 100) rounding 5.0 — NO requires condition (backward compat: always included until Phase 20 deprecates it)
- `dose_in` (18.5, 20) rounding 0.5
- `target_yield` (36, 50) rounding 1.0
- `saturation` categorical ["yes", "no"] — in "categorical" section, NO requires (backward compat)

These 6 MUST be produced by `build_parameters_for_setup("espresso", None, None)` for backward compatibility.

The ADVANCED espresso params (with `requires` conditions, for future use by Phases 20/21) are:
- `preinfusion_time` (0, 15) rounding 1.0, requires `brewer.preinfusion_type in (timed, adjustable_pressure, programmable)`
- `brew_pressure` (6, 9.5) rounding 0.5, requires `brewer.pressure_control_type in (opv_adjustable, electronic, programmable)`
- `pressure_profile` categorical ["flat", "declining", "ramp_up", "blooming"], requires `brewer.pressure_control_type in (manual_profiling, programmable)`

**Pour-over backward compatibility.** Current params:
- `grind_setting` (15, 40) rounding 0.5
- `temperature` (88, 98) rounding 1.0
- `bloom_weight` (20, 80) rounding 1.0
- `dose_in` (12, 22) rounding 0.5
- `brew_volume` (150, 500) rounding 5.0

These 5 MUST be produced by `build_parameters_for_setup("pour-over", None, None)`.

**Other 5 methods** — use the exact parameter definitions from `.planning/research/BREWING_PARAMETERS.md` section 10 (Data Model Recommendations). Use the core params only for the default (no brewer) case. Advanced params should have no `requires` for now since these methods don't have espresso-style capability dependencies — just put them in the "advanced" section.

**2. `METHOD_GRIND_PERCENTAGES` dict** — maps method to (lo_pct, hi_pct) as in research.

**3. `requires_check(condition_str, brewer)` function** — evaluates a `requires` condition string against a brewer. Pattern:
- Parse `brewer.{attr} in ({values})` format
- Return True if brewer has the attribute and its value is in the listed values
- Return True if condition is None (no requirement)
- Return True if brewer is None (backward compat — include param when no brewer specified)

**4. `build_parameters_for_setup(method, brewer=None, overrides=None)` function** — the main builder:
- Looks up method in PARAMETER_REGISTRY
- Iterates core params: include all that pass `requires_check`
- Iterates advanced params: include all that pass `requires_check`
- Iterates categorical params: include all that pass `requires_check`
- For continuous params: applies overrides (same logic as current `_resolve_bounds`), creates `NumericalContinuousParameter` with resolved bounds
- For categorical params: creates `CategoricalParameter` with OHE encoding
- For discrete params: creates `NumericalDiscreteParameter` with explicit values
- Returns list of BayBE parameter objects

**5. `get_param_columns(method, brewer=None)` function** — returns list of parameter name strings for a given method + brewer. Used by callers that need column names without building BayBE objects. When brewer is None, returns the default (backward-compatible) set.

**6. `get_default_bounds(method)` function** — returns dict of param_name -> (min, max) from registry core params. Replaces `DEFAULT_BOUNDS` and `POUR_OVER_DEFAULT_BOUNDS`.

**7. `get_rounding_rules(method)` function** — returns dict of param_name -> rounding_step from registry. Replaces `ROUNDING_RULES` and `POUR_OVER_ROUNDING_RULES`.

**8. `suggest_grind_range(grinder, method)` function** — uses `METHOD_GRIND_PERCENTAGES` + grinder.min_value/max_value to suggest grind bounds.

**IMPORTANT implementation notes:**
- Import `NumericalContinuousParameter`, `NumericalDiscreteParameter`, `CategoricalParameter` from `baybe.parameters`
- The `requires` condition evaluation should be robust — if brewer is None, all params are included (backward compat)
- For `get_default_bounds("espresso")`, it MUST return the EXACT same dict as current `DEFAULT_BOUNDS`: `{"grind_setting": (15.0, 25.0), "temperature": (86.0, 96.0), "preinfusion_pct": (55.0, 100.0), "dose_in": (18.5, 20.0), "target_yield": (36.0, 50.0)}`
- For `get_default_bounds("pour-over")`, MUST match current `POUR_OVER_DEFAULT_BOUNDS`: `{"grind_setting": (15.0, 40.0), "temperature": (88.0, 98.0), "bloom_weight": (20.0, 80.0), "dose_in": (12.0, 22.0), "brew_volume": (150.0, 500.0)}`
- `get_param_columns("espresso")` MUST return `["grind_setting", "temperature", "preinfusion_pct", "dose_in", "target_yield", "saturation"]` (exact order matches current `BAYBE_PARAM_COLUMNS`)
- `get_param_columns("pour-over")` MUST return `["grind_setting", "temperature", "bloom_weight", "dose_in", "brew_volume"]` (matches current `POUR_OVER_PARAM_COLUMNS`)
- Add a module docstring explaining the registry pattern and backward compatibility guarantees
