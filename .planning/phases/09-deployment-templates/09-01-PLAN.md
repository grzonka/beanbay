  <action>
    1. **docker-compose.yml:** Update all naming:
       - Service name: `brewflow` → `beanbay`
       - Volume name: `brewflow-data` → `beanbay-data`
       - Environment: `BREWFLOW_DATA_DIR` → `BEANBAY_DATA_DIR`
       - Add `image: ghcr.io/grzonka/beanbay:latest` so users can pull instead of build
       - Add `container_name: beanbay`
       
       Final docker-compose.yml should look like:
       ```yaml
       services:
         beanbay:
           image: ghcr.io/grzonka/beanbay:latest
           container_name: beanbay
           ports:
             - "8000:8000"
           volumes:
             - beanbay-data:/data
           environment:
             - BEANBAY_DATA_DIR=/data
           restart: unless-stopped
       
       volumes:
         beanbay-data:
       ```
    
    2. **Dockerfile:** Update env var:
       - `ENV BREWFLOW_DATA_DIR="/data"` → `ENV BEANBAY_DATA_DIR="/data"`
       - Add LABEL metadata for GHCR:
         ```dockerfile
         LABEL org.opencontainers.image.source="https://github.com/grzonka/beanbay"
         LABEL org.opencontainers.image.description="Coffee optimization powered by Bayesian learning"
         LABEL org.opencontainers.image.licenses="Apache-2.0"
         ```
    
    3. **.dockerignore:** Update to exclude non-essential files from Docker build:
       - Ensure these are excluded: `.git/`, `.planning/`, `.venv/`, `__pycache__/`, `*.pyc`, `.ruff_cache/`, `.pytest_cache/`, `tests/`, `data/`, `__marimo__/`, `my_espresso.py`, `*.egg-info/`, `.github/`, `unraid/`

<task type="auto">
  <name>Task 2: Create Unraid Community Apps XML template</name>
  <files>
    unraid/beanbay.xml
  </files>
  <action>
    Create `unraid/` directory and `beanbay.xml` template following the Unraid Community Apps template format.
    
    The XML template should include:
    
    ```xml
    <?xml version="1.0"?>
    <Container version="2">
      <Name>BeanBay</Name>
      <Repository>ghcr.io/grzonka/beanbay:latest</Repository>
      <Registry>https://github.com/grzonka/beanbay/pkgs/container/beanbay</Registry>
      <Branch>
        <Tag>latest</Tag>
        <TagDescription>Latest stable release</TagDescription>
      </Branch>
      <Network>bridge</Network>
      <Privileged>false</Privileged>
      <Support>https://github.com/grzonka/beanbay/issues</Support>
      <Project>https://github.com/grzonka/beanbay</Project>
      <Overview>
        BeanBay is a phone-first web application for optimizing coffee recipes using Bayesian optimization (BayBE). Track your beans, get AI-powered recipe recommendations, rate your brews, and watch the optimizer dial in the perfect recipe for each coffee.
        
        Self-hosted, single-user, designed for use at the espresso machine with your phone.
        
        Visit beanbay.coffee for more information.
      </Overview>
      <Category>Productivity: Tools:</Category>
      <WebUI>http://[IP]:[PORT:8000]</WebUI>
      <Icon>https://raw.githubusercontent.com/grzonka/beanbay/main/unraid/beanbay-icon.png</Icon>
      <ExtraParams/>
      <DateInstalled/>
      <Description>Coffee optimization powered by Bayesian learning. Track beans, brew, taste, optimize.</Description>
      <Config Name="Web UI Port" Target="8000" Default="8000" Mode="tcp" Description="Web interface port" Type="Port" Display="always" Required="true" Mask="false">8000</Config>
      <Config Name="Data Directory" Target="/data" Default="/mnt/user/appdata/beanbay" Mode="rw" Description="Persistent data storage (database and campaign files)" Type="Path" Display="always" Required="true" Mask="false">/mnt/user/appdata/beanbay</Config>
    </Container>
    ```
    
    Key points:
    - Repository points to GHCR image
    - WebUI uses Unraid's [IP]:[PORT:8000] syntax
    - Data maps to standard Unraid appdata location
    - Icon URL points to a file we'll create (placeholder for now — use a simple coffee bean icon or just the repo URL)
    - Category is "Productivity:" and "Tools:" (Unraid CA categories)
    - No privileged mode needed
    - Single port mapping (8000)
    - Single volume mapping (/data → appdata/beanbay)
    
    Also create a placeholder `unraid/beanbay-icon.png` note: we can't actually create a PNG programmatically. Instead, add a comment in the README about creating/adding an icon. For now, the Icon URL will 404 gracefully — Unraid shows a default icon.
  </action>

    - `python -c "import xml.etree.ElementTree as ET; ET.parse('unraid/beanbay.xml')"` succeeds
  </verify>
