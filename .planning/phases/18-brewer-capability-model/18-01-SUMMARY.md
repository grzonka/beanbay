# Phase 18 Plan 01: Brewer Capability Columns and derive_tier() Utility — Summary

**One-liner:** 13 SQLAlchemy capability columns on Brewer + idempotent Alembic migration + derive_tier() mapping capabilities to UX tiers 1-5

## What Was Built

Extended the `Brewer` model from a simple name-only record into a capability-aware entity. Added 13 new columns across 4 capability domains (temperature, pre-infusion, pressure, flow), plus two flags (has_bloom, stop_mode). Created a working Alembic migration that runs idempotently on existing databases, and a `derive_tier()` utility function that maps those capabilities to 5 UX tiers for progressive disclosure in Phase 19+.

## Decisions Made

| Decision | Why |
|----------|-----|
| Add `__init__` to Brewer with Python-side defaults | `Column(default=...)` only applies at DB INSERT time. Plan verify step expects pre-flush access (`Brewer(name='x').temp_control_type == 'pid'`), so `__init__` with `setdefault()` was added. Consistent with what the plan specified. |
| Excluded `measurements.flavor_tags` type change from migration | Autogenerate detected `TEXT()` vs `String()` difference — semantically identical in SQLite. Keeping it would create a confusing no-op change in migration history. |

## Capability Model Design

### 13 Columns Added to `Brewer`

| Column | Type | Default | Domain |
|--------|------|---------|--------|
| `temp_control_type` | String | `"pid"` | Temperature |
| `temp_min` | Float | None | Temperature |
| `temp_max` | Float | None | Temperature |
| `temp_step` | Float | None | Temperature |
| `preinfusion_type` | String | `"none"` | Pre-infusion |
| `preinfusion_max_time` | Float | None | Pre-infusion |
| `pressure_control_type` | String | `"fixed"` | Pressure |
| `pressure_min` | Float | None | Pressure |
| `pressure_max` | Float | None | Pressure |
| `flow_control_type` | String | `"none"` | Flow |
| `has_bloom` | Boolean | `False` | Misc |
| `stop_mode` | String | `"manual"` | Misc |

### Module Constants

```python
TEMP_CONTROL_TYPES     = ("none", "preset", "pid", "profiling")
PREINFUSION_TYPES      = ("none", "fixed", "timed", "adjustable_pressure", "programmable", "manual")
PRESSURE_CONTROL_TYPES = ("fixed", "opv_adjustable", "electronic", "manual_profiling", "programmable")
FLOW_CONTROL_TYPES     = ("none", "manual_paddle", "manual_valve", "programmable")
STOP_MODES             = ("manual", "timed", "volumetric", "gravimetric")
```

### Tier Derivation Logic

| Tier | Label | Trigger |
|------|-------|---------|
| 5 | Full Programmable | `flow_control_type == "programmable"` |
| 4 | Pressure & Flow | `flow_control_type in (manual_paddle, manual_valve)` OR `pressure_control_type in (manual_profiling, programmable)` |
| 3 | Pre-infusion | `preinfusion_type in (timed, adjustable_pressure, programmable)` |
| 2 | Temperature Control | `temp_control_type in (pid, profiling)` |
| 1 | Basic | fallthrough |

**Real machine examples:** Gaggia Classic stock → T1, Sage Dual Boiler → T3, Decent DE1 → T5.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Column(default=...) doesn't provide Python-side attribute defaults**

- **Found during:** Task 1 verification
- **Issue:** `Brewer(name='Test').temp_control_type` returned `None` instead of `"pid"` because `Column(default=...)` only fires at INSERT time
- **Fix:** Added `__init__` method to `Brewer` that calls `kwargs.setdefault(...)` for all non-nullable columns, then delegates to `super().__init__(**kwargs)`
- **Files modified:** `app/models/equipment.py`
- **Commit:** `a24185b`
