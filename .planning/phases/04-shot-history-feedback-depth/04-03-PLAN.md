  **Add endpoints to `app/routers/history.py`:**

  ```
  GET /history/{shot_id} — Shot detail (returns modal HTML)
    - Load measurement by ID, join with Bean for name
    - Parse flavor_tags from JSON to list
    - Compute brew ratio
    - Render _shot_modal.html
    - Response includes an HX-Trigger header: "openShotModal" (used by JS to call dialog.showModal())

  GET /history/{shot_id}/edit — Shot edit form (returns edit form HTML inside modal)
    - Same data loading as detail
    - Render _shot_edit.html (with form pre-populated)
    - The edit form includes the feedback panel fields (notes, flavor sliders, tags) pre-populated with existing values

  POST /history/{shot_id}/edit — Save edits
    - Form params: notes, acidity, sweetness, body, bitterness, aroma, intensity, flavor_tags
    - Update the measurement record
    - For flavor dimensions: if value submitted, update; if empty/missing, set to None (user cleared it)
    - For flavor_tags: parse comma-separated string to JSON list, save
    - For notes: save (empty string → None)
    - Return TWO things via htmx oob:
      1. Updated _shot_modal.html as the main response (replaces modal content)
      2. Updated _shot_row.html with hx-swap-oob="true" targeting #shot-{id} (updates the list row behind the modal)
    - To do oob: render _shot_modal.html as main content + append rendered _shot_row.html with `hx-swap-oob="outerHTML:#shot-{shot_id}"` attribute
  ```
