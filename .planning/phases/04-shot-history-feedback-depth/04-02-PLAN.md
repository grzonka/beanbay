  <files>app/routers/history.py, app/main.py, app/templates/base.html, app/templates/beans/detail.html</files>
  <action>
  **Create `app/routers/history.py`:**
  ```
  Router prefix: /history, tags: ["history"]

  GET /history — Full history page
    - Query params: bean_id (optional), min_taste (optional float)
    - Load all beans for filter dropdown
    - Load active_bean from cookie (for nav display)
    - If bean_id query param present, pre-select that bean in filter
    - Render history/index.html with beans list, active filters, shot list
    - Shot query: all measurements, reverse chronological (created_at DESC)
    - Apply filters: if bean_id, filter by bean_id; if min_taste, filter taste >= min_taste
    - Join with Bean to get bean name for each shot row
    - Return full page

  GET /history/shots — htmx partial for filtered shot list
    - Same query params: bean_id, min_taste  
    - Same filtering logic
    - Return only _shot_list.html partial (for htmx swap)
    - Check HX-Request header to confirm htmx call
  ```

  Use the same `_get_active_bean` helper pattern from beans.py/brew.py (read from cookie). Import Bean and Measurement models. Use `db.query(Measurement).join(Bean)` to get bean name alongside measurement data.

  For each measurement, compute brew ratio using dose_in/target_yield. Parse flavor_tags from JSON string to list for display (use `json.loads` with fallback to empty list).

  **Wire router in `app/main.py`:**
  - Add `from app.routers import history` (alongside existing beans, brew imports)

  - This pre-applies the bean filter when navigating from bean detail
