### Task 1: Create SimilarityService

**File:** `app/services/similarity.py`

Create a service that finds similar beans by process + variety.

**Requirements:**
- `SimilarBean` dataclass: `bean_id: str`, `bean_name: str`, `process: str | None`, `variety: str | None`, `measurement_count: int`
- `find_similar_beans(target_bean: Bean, method: str, db: Session, min_measurements: int = 3) -> list[SimilarBean]`
  - Returns beans (excluding target) where BOTH process AND variety match target bean
  - Only returns beans where process is not None AND variety is not None (can't match on missing data)
  - Only returns beans with `>= min_measurements` measurements for the given method
    - Method match: look at `brew_setup.brew_method.name` on measurements with `brew_setup_id` set, OR count legacy espresso measurements (no brew_setup_id, treat as espresso)
  - Returns empty list if target bean has no process or no variety
  - Orders by measurement_count DESC (most data first)
- `count_method_measurements(bean_id: str, method: str, db: Session) -> int`
  - Count measurements for a bean+method combination
  - Espresso measurements: those with NULL brew_setup_id OR brew_setup.brew_method.name == "espresso"
  - Pour-over: those with brew_setup.brew_method.name == "pour-over"

**Done criteria:**
- `SimilarityService` class exists in `app/services/similarity.py`
- All methods implemented as specified
- No imports outside stdlib + sqlalchemy + app.models

**Verification:**

### Task 3: Create TransferLearningService

**File:** `app/services/transfer_learning.py`

Build the service that creates a TaskParameter-seeded BayBE campaign from similar beans.

**Requirements:**

```python
from dataclasses import dataclass
from baybe.campaign import Campaign
from app.models import Bean, Measurement
from app.services.optimizer import _create_fresh_campaign  # will refactor to static or module-level
from app.services.similarity import SimilarBean

@dataclass
class TransferMetadata:
    contributing_beans: list[dict]  # [{"bean_id": ..., "name": ..., "process": ..., "variety": ...}]
    total_training_measurements: int
```

**`build_transfer_campaign(target_bean, similar_beans, method, overrides, db) -> tuple[Campaign, TransferMetadata] | None`**

Logic:
1. If `similar_beans` is empty: return `None`
2. Collect training measurements from each similar bean for the given method:
   - Espresso: query Measurement where bean_id = similar.bean_id AND (brew_setup_id IS NULL OR join brew_setup â†’ brew_method.name == "espresso")
   - Pour-over: similar but brew_method.name == "pour-over"
   - Get only BayBE-relevant columns + "taste"
3. Build all task labels: `[similar.bean_id for similar in similar_beans] + [target_bean.id]`
4. Create campaign with `TaskParameter`:
   ```python
   TaskParameter(name="bean_task", values=all_task_ids, active_values=[target_bean.id])
   ```
   Plus the standard method parameters (same as `_create_fresh_campaign` but with TaskParameter added)
5. Add training measurements to campaign (each row needs `bean_task = similar.bean_id`)
6. Return (campaign, TransferMetadata)

**Note on `_create_fresh_campaign`:** Extract the parameter-building logic into a module-level
helper `_build_parameters(overrides, method) -> list` so both `_create_fresh_campaign` and
`build_transfer_campaign` can use it without code duplication.

**Done criteria:**
- `TransferLearningService` (or module-level function `build_transfer_campaign`) in `app/services/transfer_learning.py`
- `TransferMetadata` dataclass exported
- Returns `None` when `similar_beans` is empty
- Campaign includes `TaskParameter` as first parameter
- Training measurements include `bean_task` column

**Verification:**
