# Phase 16, Plan 02: Wire Transfer Learning into Optimizer

## Objective

Update `OptimizerService` to automatically use `TransferLearningService` when creating a
new campaign for a bean that has similar beans in history. Store transfer metadata as a
`.transfer` JSON file alongside the campaign. Update the brew router to pass the bean + db
session when calling `recommend()`, and read the `.transfer` file to pass transfer context
to the template.

## Context Files

- @app/services/optimizer.py
- @app/services/transfer_learning.py
- @app/services/similarity.py
- @app/routers/brew.py
- @app/templates/brew/recommend.html
- @.planning/phases/16-transfer-learning/16-CONTEXT.md

### Task 1: Update OptimizerService to use transfer learning on campaign creation

**File:** `app/services/optimizer.py`

**Changes to `get_or_create_campaign`:**

Add two optional parameters:
```python
def get_or_create_campaign(
    self,
    campaign_key: str,
    overrides: dict | None = None,
    method: str = "espresso",
    target_bean=None,   # Bean ORM object, optional
    db=None,            # SQLAlchemy Session, optional
) -> Campaign:
```

In the "create fresh" branch (when `path.exists()` is False):
1. If `target_bean` is not None AND `db` is not None:
   - Call `SimilarityService().find_similar_beans(target_bean, method, db)`
   - If similar beans found: call `build_transfer_campaign(target_bean, similar_beans, method, overrides, db)`
   - If `build_transfer_campaign` returns non-None: use that campaign, write `.transfer` metadata file
2. Otherwise (or if no similar beans): use `_create_fresh_campaign(overrides, method)` as before

**Transfer metadata file:**
- Path: `{campaign_key}.transfer` alongside `{campaign_key}.json`
- Content: JSON of `TransferMetadata` fields
  ```json
  {
    "contributing_beans": [{"bean_id": "...", "name": "...", "process": "...", "variety": "..."}],
    "total_training_measurements": 6
  }
  ```

**Add helper methods:**
```python
def get_transfer_metadata(self, campaign_key: str) -> dict | None:
    """Return transfer metadata dict if this campaign was seeded via transfer learning."""
    path = self._campaigns_dir / f"{campaign_key}.transfer"
    if path.exists():
        return json.loads(path.read_text())
    return None
```

**Update `add_measurement` to handle transfer learning campaigns:**
When a campaign has a `.transfer` file, the campaign's search space includes `TaskParameter`.
Measurements must include `bean_task = target_bean_id`. Update `add_measurement` signature:
```python
def add_measurement(
    self,
    campaign_key: str,
    measurement: dict,
    overrides: dict | None = None,
    method: str = "espresso",
    target_bean_id: str | None = None,  # NEW: required for transfer learning campaigns
) -> None:
```
If `get_transfer_metadata(campaign_key)` is not None AND `target_bean_id` is provided:
- Add `bean_task = target_bean_id` to `baybe_data` before adding to campaign

**Done criteria:**
- `get_or_create_campaign` accepts `target_bean` and `db` optional params
- Transfer metadata file written when transfer learning activates
- `get_transfer_metadata` method implemented
- `add_measurement` handles `bean_task` column for transfer learning campaigns
- All existing optimizer tests still pass
