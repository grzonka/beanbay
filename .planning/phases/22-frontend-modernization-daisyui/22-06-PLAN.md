**Step 2: Prune unused custom CSS from input.css.**

For each custom class defined in input.css's `@layer components`, verify at least one template references it:
- Run `grep -rl "className" app/templates/` for each class
- If a class is defined in input.css but NO template uses it, remove it
- This is unlikely (input.css was designed to match template usage) but verify

**Step 3: Rebuild CSS.**

Run `make css` (or the equivalent Tailwind CLI command) to compile input.css → main.css:
```bash
make css
```

If `make css` is not set up yet (depends on Plan 01 completion), run directly:
```bash
./tools/tailwindcss -i app/static/css/input.css -o app/static/css/main.css --minify
```

Verify the compiled output:
- `main.css` should be significantly smaller than the old 1935-line version
- It should contain daisyUI component CSS + Tailwind utilities + custom layer CSS
- It should NOT contain any of the old hand-rolled CSS classes

**Step 4: Run the full test suite.**

```bash
python -m pytest tests/ -v
```

All tests must pass. The test suite tests functionality (routes, models, API behavior), not CSS — so they should pass without modification. If any test fails, investigate:
- Template rendering errors (missing Jinja2 variable, broken include)
- Route errors (not CSS-related, likely pre-existing)
