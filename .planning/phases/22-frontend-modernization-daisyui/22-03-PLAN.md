This is the most complex single template (310 lines). It has: collapsible sections, 5 modal overlays (grinder-add, brewer-add, paper-add, water-add, equipment-edit), and section headers with badges.

**1. Collapsible sections — use daisyUI `collapse` component:**

Replace the current `<button class="collapsible-toggle">` + `<div class="collapsible-content">` pattern with daisyUI's details-based collapse:

```html
<div class="collapse collapse-arrow bg-base-200 border border-base-300 mb-2">
  <input type="checkbox" checked />  {# checked = starts open; omit for closed #}
  <div class="collapse-title font-semibold flex items-center gap-2">
    Brew Setups
    <span class="badge badge-ghost">{{ setup_count }}</span>
  </div>
  <div class="collapse-content">
    {# ... content ... #}
  </div>
</div>
```

For the "Brew Setups" section: start open (`checked`). For Grinders, Brewers, Papers, Water: start closed (no `checked`).

**Note:** The current collapsible uses `onclick="this.nextElementSibling.classList.toggle('open')"` — this JS is no longer needed with daisyUI's checkbox-based collapse. Remove all `onclick` handlers from collapsible toggles.

**IMPORTANT:** The collapse-arrow uses `<input type="checkbox">` — NOT `<details>/<summary>`. Using checkbox input allows multiple sections to be open simultaneously (details/summary doesn't allow this by default). The checkbox pattern is the correct daisyUI approach.

**2. Convert equipment modals from custom overlay to daisyUI `<dialog>` modal:**

**Old pattern (custom overlay):**
```html
<div class="equipment-modal-overlay" id="grinder-add-modal" onclick="closeEquipmentModalOnOverlay(event, 'grinder-add-modal')">
  <div class="equipment-modal-box">
    <div class="equipment-modal-header">
      <h2 class="equipment-modal-title">Add Grinder</h2>
      <button class="equipment-modal-close" onclick="closeEquipmentModal('grinder-add-modal')">✕</button>
    </div>
    {% include "equipment/_grinder_form.html" %}
  </div>
</div>
```

**New pattern (daisyUI `<dialog>`):**
```html
<dialog id="grinder-add-modal" class="modal">
  <div class="modal-box bg-base-200 border border-base-300 max-w-lg">
    <div class="flex items-center justify-between mb-4 pb-2 border-b border-base-300">
      <h2 class="text-lg font-bold">Add Grinder</h2>
      <form method="dialog"><button class="btn btn-ghost btn-sm btn-square">&#10005;</button></form>
    </div>
    {% set grinder = None %}
    {% include "equipment/_grinder_form.html" %}
  </div>
  <form method="dialog" class="modal-backdrop"><button>close</button></form>
</dialog>
```

Convert ALL 5 modals to this pattern:
- `grinder-add-modal`
- `brewer-add-modal`
- `paper-add-modal`
- `water-add-modal`
- `equipment-edit-modal`

**3. Update JavaScript functions:**

Replace the old overlay-based JS with native dialog methods:

```javascript
function openEquipmentModal(id) {
  document.getElementById(id).showModal();
}

function closeEquipmentModal(id) {
  document.getElementById(id).close();
}
```

**Remove** `closeEquipmentModalOnOverlay` — the `<form method="dialog" class="modal-backdrop">` handles click-outside-to-close natively.

**Keep** `openEditModal(url)` — update to use `showModal()` instead of `classList.add('open')`:

```javascript
function openEditModal(url) {
  fetch(url, { headers: { 'HX-Request': 'true' } })
    .then(r => r.text())
    .then(html => {
      document.getElementById('equipment-modal-content').innerHTML = html;
      document.getElementById('equipment-edit-modal').showModal();
    });
}
```

**Keep** the `htmx:afterSwap` event listener for removing empty states — update card class selectors if class names changed (e.g., `grinder-card` → check if cards still use this class).

**4. Section styling:**
- Page header: `<div class="flex items-center justify-between mb-6 min-h-12">`
- Page title: `<h1 class="text-xl font-bold">`
- Show retired toggle: `<label class="flex items-center gap-2 cursor-pointer text-sm text-base-content/70"><input type="checkbox" class="checkbox checkbox-sm" ...>Show retired</label>`
- "Add" buttons inside sections: `<button class="btn btn-ghost btn-sm mb-2">+ Add Grinder</button>`
- Empty states: `<div class="text-center py-6 text-base-content/50">`

**5. Remove ALL inline `style=` attributes** — replace with Tailwind utilities:
- `style="padding-top: var(--spacing-sm);"` → Tailwind class on parent or `pt-2`
- `style="margin-bottom: var(--spacing-sm);"` → `mb-2`
- `style="display:inline-block; margin-bottom: var(--spacing-sm);"` → `inline-block mb-2`

**CRITICAL — Preserve:**
