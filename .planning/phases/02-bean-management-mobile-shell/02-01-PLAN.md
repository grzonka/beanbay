# Plan: Mobile Shell & Bean CRUD

**Phase:** 2 — Bean Management & Mobile Shell
**Plan:** 02-01
**Goal:** Set up the Jinja2 + htmx mobile-first shell and implement bean CRUD API endpoints with active bean selection.

## Context

Phase 1 delivered the FastAPI skeleton, SQLAlchemy models, and BayBE service. The app currently returns JSON from `/` and `/health`. Phase 2 transforms this into a real mobile-first web app.

The stack decision from research: Jinja2 templates + htmx for interactivity + vanilla CSS for mobile-first responsive layout. No frontend build step.

## Tasks

### 1. Jinja2 Template Setup
- Add `Jinja2Templates` initialization in `app/main.py`
- Create `app/templates/base.html` — the mobile-first shell:
  - HTML5 doctype with viewport meta for mobile
  - htmx CDN script tag (v2.x)
  - CSS either inline in base or in `app/static/css/main.css`
  - Mobile-first CSS: max-width container, 48px min touch targets, generous spacing
  - Navigation: simple top bar with app name + active bean indicator
  - Content block for page-specific content
  - 375px as the primary design width (phone at espresso machine)

### 2. Mobile-First CSS (`app/static/css/main.css`)
Design tokens / rules:
- `--touch-target: 48px` minimum for all interactive elements
- `--spacing-sm: 8px`, `--spacing-md: 16px`, `--spacing-lg: 24px`
- Font sizes: body 16px (prevents iOS zoom), headings proportional
- `.btn` class: full-width on mobile, 48px+ height, large text
- `.input` class: full-width, 48px+ height, 16px font (prevents iOS zoom)
- `.card` class: for bean list items, full-width, generous padding
- Color scheme: dark background for espresso vibe, high contrast text
- System font stack (fast, native feel)

### 3. Bean CRUD API Endpoints
Create `app/routers/beans.py` with APIRouter:

- `GET /beans` — List all beans with shot counts
  - Returns HTML page (Jinja2 template) with bean list
  - Each bean shows: name, roaster, shot count, active indicator
  - "Add Bean" button at top
  
- `POST /beans` — Create a new bean
  - Form fields: name (required), roaster (optional), origin (optional)
  - htmx: returns the new bean card fragment to append to list
  - Redirects to bean list on non-htmx request

- `GET /beans/{bean_id}` — Bean detail / edit page
  - Shows bean info + parameter overrides form
  - "Select as Active" button
  - Shot count

- `PUT /beans/{bean_id}` — Update bean (name, roaster, origin, parameter_overrides)
  - htmx: returns updated bean detail fragment

- `DELETE /beans/{bean_id}` — Delete bean + cascade measurements
  - htmx: removes bean from list

- `POST /beans/{bean_id}/activate` — Set as active bean
  - Stores active_bean_id in a cookie (no auth, single-user)
  - htmx: updates active indicator in nav + bean list

### 4. Active Bean Cookie Middleware
- Simple cookie-based active bean tracking: `active_bean_id`
- Middleware or dependency that reads the cookie and makes it available to templates
- No authentication needed (single-user home app)

### 5. Update Root Route
- `GET /` → redirect to `/beans` (bean list is the home page)

## Verification

- `GET /` redirects to `/beans`

- Creating a bean via POST works and the bean appears in the list
- Activating a bean sets the cookie and shows in the nav
- All mobile CSS rules enforced: 48px touch targets, 375px primary width
- No JavaScript build step required (htmx from CDN, vanilla CSS)
