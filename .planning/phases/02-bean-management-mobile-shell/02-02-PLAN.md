# Plan: Tests & Parameter Overrides UI

**Phase:** 2 — Bean Management & Mobile Shell
**Plan:** 02-02
**Goal:** Add comprehensive tests for bean CRUD + mobile UI, and build the parameter overrides editing UI.

## Context

Plan 02-01 delivers the mobile shell and bean CRUD endpoints. This plan adds tests to verify everything works and builds the parameter overrides form for per-bean BayBE bound customization (feature implemented in the optimizer layer, needs UI).

## Tasks

### 1. Bean CRUD Tests (`tests/test_beans.py`)
Using FastAPI TestClient + httpx:

- `test_list_beans_empty` — GET /beans returns page with empty state
- `test_create_bean` — POST /beans creates a bean, appears in list
- `test_create_bean_name_required` — POST /beans without name fails
- `test_bean_detail` — GET /beans/{id} shows bean info
- `test_update_bean` — PUT /beans/{id} updates fields
- `test_delete_bean` — DELETE /beans/{id} removes bean
- `test_activate_bean` — POST /beans/{id}/activate sets cookie
- `test_active_bean_shown_in_nav` — active bean name appears in nav
- `test_bean_list_shows_shot_count` — beans show measurement count

### 2. Parameter Overrides UI
In the bean detail page (`app/templates/beans/detail.html`):

- Collapsible "Custom Ranges" section (collapsed by default)
- For each overridable parameter (grind, temp, preinfusion%, dose, yield):
  - Show current range (default or custom)
  - Min/max number inputs
  - "Reset to default" button per parameter
- Form submits via htmx PUT to `/beans/{bean_id}`
- Server-side validation: min < max, values within reasonable absolute limits
- On save: campaign is automatically invalidated (handled by optimizer service)

### 3. Parameter Override Validation
In `app/routers/beans.py`:

- Validate override values: min < max
- Validate absolute limits (e.g., temperature can't be negative)
- Return clear error messages for invalid input

### 4. Mobile UI Polish
- Test on 375px viewport (Chrome DevTools)
- Verify all touch targets are 48px+
- Verify no horizontal scroll
- Verify form inputs don't trigger iOS zoom (16px+ font size)

## Verification

- All tests pass: `pytest tests/ -v`
- Bean CRUD works end-to-end through the UI
- Parameter overrides form saves and loads correctly
- Changing overrides triggers campaign rebuild (verified in optimizer tests already)
- Mobile layout works at 375px width
